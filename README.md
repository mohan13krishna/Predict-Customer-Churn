## 🏢 **CHURN PREDICTION ARSENAL - THE 5+ MODEL STRATEGY**

### **🥇 PRIMARY: XGBoost (Gradient Boosting Excellence)**
- **Framework:** Fast, GPU-optimized XGBoost
- **Architecture:** lr=0.0063 (fixed, bi-optimized), max_depth=5, colsample_bytree=0.32
- **Role:** The BACKBONE of our ensemble
- **Expected Strength:** Best individual performance (CV: 0.91796)
- **Key Params:** early_stopping_rounds=500, min_child_weight=6

### **🥈 SECONDARY: LightGBM (Efficient Gradient Boosting)**
- **Framework:** Fast, memory-efficient tree learning
- **Architecture:** max_depth=7, num_leaves=55, lr=0.02706
- **Role:** The SPEED instrument
- **Expected Strength:** Quick iteration, consistent results (CV: 0.91781)
- **Key Params:** early_stopping_rounds=200, min_child_samples=71

### **🥉 TERTIARY: CatBoost (Categorical Specialist)**
- **Framework:** Native categorical feature handling
- **Architecture:** depth=5, lr=0.05-0.13171 (variable), iterations=50000
- **Role:** The CATEGORICAL expert
- **Expected Strength:** Superior category understanding (CV: 0.91767)
- **Key Params:** auto_class_weights='Balanced', early_stopping_rounds=100

### **💪 SUPPORT: Entity Embedding MLP (Neural Network)**
- **Framework:** PyTorch deep learning
- **Architecture:** Embedding layer per categorical column + 4-layer dense network
- **Role:** The DEEP LEARNING diversity
- **Expected Strength:** Non-linear patterns (CV: 0.91705)
- **Key Params:** OneCycleLR scheduler, BatchNorm + GELU + Dropout(0.3)

### **🔥 RESERVE: Plain MLP (PyTorch Neural Network)**
- **Framework:** Standard fully-connected neural network
- **Architecture:** 512→256→128→64→1 with BatchNorm + GELU
- **Role:** The ARCHITECTURE diversity
- **Expected Strength:** Fast neural baseline (CV: 0.91700)
- **Key Params:** AdamW optimizer, CosineAnnealingLR scheduler

### 🔄 **THE JOURNEY: DATA → COMPETITION SUBMISSION**

#### **Phase 1: Data Foundation** 🏗️
- Load combined 601K samples with optimal preprocessing

- Missing value handling (TotalCharges imputation strategy)

- **Static Features** (~250-269 columns): OHE, ordinal encoding, numeric transforms
- **Target-Encoded Features** (~52 per fold): 
  - 16 categorical base × 3 statistics (mean, count_norm, std)
  - 4 trigram pairs × 1 statistic (single TE per trigram)
  - Total TE per fold: 52 features
- **Trigram Expansion** (V22+): 20 trigram pairs vs 4 base (C(6,3) combinations)
- **Advanced Statistics**: Median, Skew added to ORIG features

#### **Phase 3: Base Model Development** 💡
- Individual training: XGBoost, LightGBM, CatBoost
- Hyperparameter optimization via grid/random search
- Cross-validation strategy: 20-fold stratified (deeper than typical)
- Pseudo-labeling with TRES=0.999 threshold

#### **Phase 4: Neural Network Diversity** 🧠
- **V24 (PlainMLP)**: 4-layer MLP, dropout=0.3, AdamW optimizer
- **V25 (EntityEmbeddingMLP)**: Category embeddings + numeric branch

- **V21**: Multi-seed XGB averaging (SEED=42, 123, 456) → 3-seed diversity

#### **Phase 6: BLENDING & ENSEMBLE OPTIMIZATION** 👑

## 💡 EXPERT STRATEGIES THAT WORK

### ✅ **What Powers Our Approach**

1. **Multi-Seed Diversity Strategy** 🌱
   - SEED=42 (V38): Baseline ensemble
   - SEED=123 (V41): Secondary diversity
   - SEED=456 (V26→V56): Tertiary diversity for 3-way blend
   - Result: 60 independent fold models (20 folds × 3 seeds)

2. **Deep Cross-Validation** 🔄
   - 20-fold stratified CV (vs typical 5-fold)
   - Provides more OOF predictions for meta-learning
   - Better generalization through multiple train/val splits

3. **Target Encoding TE Innovation** 📊
   - Per-fold target encoding with smoothing
   - Avoids target leakage through fold-aware preprocessing
   - 52 TE features per fold = strong signal

4. **Neural Network Diversity** 🧠
   - PlainMLP: Fast, standard dense architecture
   - EntityEmbeddingMLP: Preserves category structure
   - Both trained with OneCycleLR/CosineAnnealingLR
   - Expected contribution: 2-3% ensemble weight

5. **Rank-Based Blending** 📈
   - Normalize to [0,1] via rank transformation
   - More robust than raw blending to outliers
   - Used in all V44-V59 submissions

6. **Hill Climbing Optimization** 🎯
   - Random weight sampling (3000-5000 iterations)
   - Maximize OOF AUC via weight tuning
   - Expected best weights: [XGB~0.50, LGBM~0.35, CAT~0.15...]

## 📊 BLENDING STRATEGY BREAKDOWN

### **Rank Normalization Formula**
```
All blends use rank-based normalization:
1. For each model's OOF: rank = rankdata(oof_predictions)
2. Normalize: rank_normalized = rank / (len(rank) + 1)
3. Weighted blend: final = Σ(weight_i × rank_normalized_i)
4. Result: Robust to outliers, preserves monotonicity
```
